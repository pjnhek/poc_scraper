<objective>
Ship `scripts/verify_public_repo.py` and a `make verify-public-repo` Makefile target that re-verify the public-repo audit (REPO-01, REPO-03) on demand. Purpose: Formalize the verification surface for the Phase 7 audit so the demo SHA is provably clean before Phase 8 records the Loom. Mirrors the existing `scripts/check_public_discipline.py` shape (strict-mypy clean, same DENYLIST path, same subprocess style).

Output: One new Python script + one new Makefile target. Both are operator-triggered; not wired into CI (per CONTEXT.md deferred: CI shallow clones cannot scan `git log --all -p`).

Create a new file `scripts/verify_public_repo.py` that mirrors `scripts/check_public_discipline.py` strict-mypy shape (full annotations, `from __future__ import annotations`, module docstring explaining purpose plus denylist-file framing plus usage line).

Module-level constant: `DENYLIST = Path(__file__).resolve().parent.parent / ".secrets-denylist"`. Reuse the exact `_load_patterns()` helper verbatim from `scripts/check_public_discipline.py:21-30` (return type `list[re.Pattern[str]]`; per D-Claude's-Discretion the planner explicitly chose to duplicate the 5-line helper rather than extract to `scripts/_denylist.py` — DRY cost is low and both scripts already exist).

Implement two pure helper functions plus a `main() -> int`:

1. Worktree pass — list tracked file paths via `subprocess.run(["git", "ls-files"], capture_output=True, text=True)`. Iterate paths. For each path: skip if path equals `.secrets-denylist` per D-03 (the one allowed location; gitignored anyway but worktree scan still sees it). Read the file body if it is a text file (`open(path, encoding="utf-8", errors="replace").read()`); for each compiled pattern, count occurrences with `len(pat.findall(content))` and ALSO `len(pat.findall(path))` so the count reflects true occurrences (not just a boolean match). Increment a worktree-occurrence counter by the per-call sum. Do NOT collect raw match text. Do NOT print the path or pattern to stdout per D-11 and THR-01. Skip binary files defensively via a `try/except (UnicodeDecodeError, OSError)` block that increments a "skipped" counter the script does not surface (silent skip is fine; the goal is text-content scanning).

2. History pass — `subprocess.run(["git", "log", "--all", "-p"], capture_output=True, text=True, check=False)`. Iterate `result.stdout.splitlines()`. For each line, for each compiled pattern, increment the history counter by `len(pat.findall(line))`. The full diff blob is large but the operator runs this rarely; an in-process iteration is fine. Do NOT print matched lines.

3. `main() -> int`:
   - If not `DENYLIST.exists()`: write to stderr `"verify-public-repo: .secrets-denylist not found at <path>. Create it with one regex per line (see README Local setup) before running."` and `return 2` per D-08.
   - Else call `_load_patterns()`; if the list is empty (denylist exists but contains only comments / blanks), write to stderr `"verify-public-repo: .secrets-denylist contains no active patterns; nothing to verify."` and `return 2` (treat as a setup error, not a clean pass — the operator clearly intended to configure patterns).
   - Run worktree pass, then history pass. Compute counts.
   - Resolve current HEAD SHA for the summary line: `subprocess.run(["git", "rev-parse", "--short", "HEAD"], capture_output=True, text=True, check=False).stdout.strip()`. If empty, use the literal string `"unknown"`.
   - Print to stdout exactly one summary line: `f"verify-public-repo: {worktree_hits} hits in tracked content, {history_hits} hits in history reachable from any ref (commit {sha})."` where the two integer counts are true occurrence counts produced by `len(pat.findall(...))` (per I2 in the revision context: counts reflect occurrences, not boolean file-matches). This is the only stdout output; it contains counts and a SHA only, never raw match text.
   - If `worktree_hits == 0 and history_hits == 0`: return 0.
   - Else: write to stderr a single non-revealing line `"verify-public-repo: non-zero hits found; inspect .secrets-denylist patterns and grep locally to remediate. Do NOT paste hit text into any committed artifact."` and return 1 per D-Claude's-Discretion exit-code contract. (Note: there is no `--verbose` flag implemented or planned in this iteration; the stderr message must not reference one.)

The script accepts NO argv flags in this iteration. Do not introduce, document, or hint at flag-based affordances; the missing-flag stays silent in the codebase rather than appearing as a stale breadcrumb in stderr.

Entry point: `if __name__ == "__main__": raise SystemExit(main())`.

Do NOT add ripgrep / `rg` as a tool dependency; pure-Python `re.search` over `git ls-files` and `git log --all -p` is the chosen implementation per the Read-first analog and avoids carrying a binary dep. CONTEXT.md D-01 says ripgrep "is acceptable" but PATTERNS.md notes the project does not carry it; chosen path: stdlib only.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| script output -> committed artifacts | The verify script's stdout / stderr feeds into the operator's terminal and from there potentially into FINDINGS.md (Plan 3). Anything the script prints could be quoted by a future maintainer. Output must therefore contain ZERO raw matches; counts and a SHA only. |
| operator filesystem -> repo | The script reads `.secrets-denylist` (gitignored, local-only). The denylist itself never enters the repo; the script must not echo its contents either. |
| script scan boundary | The script's "what is clean" surface is the current working tree + its git history. Anything outside (e.g. `../poc_scraper-FULL-BACKUP.bundle`) is intentionally excluded per D-17 to keep the script's contract simple and to avoid coupling Phase 7 verification to Phase 8 bundle timing. |

| Threat ID | Category | Component | Disposition | Mitigation Plan |

Stdout emits exactly one summary line with counts + SHA. Stderr emits non-revealing setup messages only. The "do NOT paste hit text into any committed artifact" stderr message exists specifically to remind the operator on a non-zero exit.

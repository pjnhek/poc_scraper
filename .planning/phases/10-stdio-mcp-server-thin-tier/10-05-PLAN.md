## Trust Boundaries

| Boundary | Description |
|---|---|
| Exa/Browserbase -> RawContext -> MCP client | Untrusted provider provenance is filtered, capped, numbered, serialized, and placed into the calling agent's context. |
| MCP client -> Account -> Exa | Untrusted domain text can trigger paid provider calls or inject malformed authority syntax unless rejected before lifespan access. |
| Account validation -> FastMCP error result -> MCP client | Validation detail becomes client-visible tool text and can amplify attacker-controlled input unless the message is bounded. |

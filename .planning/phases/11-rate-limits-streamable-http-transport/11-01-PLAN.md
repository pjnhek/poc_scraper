Internal state: a dict of per-IP `deque[datetime]` buckets (deque bounded via maxlen=ip_limit as defense-in-depth; the length check happens before append so maxlen never actually evicts), a current-UTC-day marker, a day counter, and an `asyncio.Lock`.
    - `async def check_and_consume(self, ip: str) -> LimitResult`: the single check-and-consume point per D-04. Everything runs inside `async with self._lock:` and the body contains NO await points (the lock is defense-in-depth per RESEARCH.md Pitfall 5: it protects the read-modify-write invariant against future edits that introduce an await; add a short WHY comment saying exactly that). Sequence inside the lock: read `now = self.clock()`; roll the day counter if `now.astimezone(UTC).date()` differs from the stored day; prune the IP's bucket from the left while the oldest entry is at or older than now minus one hour, deleting the dict key when a bucket prunes empty (memory hygiene); if the pruned bucket length is at ip_limit, return a refused LimitResult whose message is the string `rate limit reached, resets at ` followed by the oldest remaining timestamp plus one hour rendered with strftime format `%H:%M` in UTC and the suffix ` UTC` (D-06: plain message, carries reset time); else if the day counter is at daily_cap, return refused with DAILY_CAP_MESSAGE; else append now to the bucket, increment the day counter, and return allowed. Per-IP is checked before global so a doubly-exhausted caller sees the per-IP message; a refusal by either check happens before any mutation, so neither counter is consumed (D-04).
    - `resolve_client_ip(request: Request | None) -> str`: takes a `starlette.requests.Request | None` (NOT a Context; keeps this module MCP-free and unit-testable with bare Request objects). None returns SHARED_BUCKET. If absent, read `x-forwarded-for`, take the rightmost comma-separated entry via `rsplit(",", 1)[-1].strip()`, validate the same way, return canonical form or SHARED_BUCKET. Never log raw header values (they are untrusted input; this function should not log at all).

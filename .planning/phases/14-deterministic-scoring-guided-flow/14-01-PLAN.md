<objective>
Add the deterministic `score_account` MCP tool: a pure-arithmetic wrapper over the existing `compute_total`/`verdict_for` math, registered on both tiers, with the project's sanitized-error discipline and full unit + in-memory-MCP-client test coverage.

Implements locked decisions D-01 (integer-only axis scores 1-5), D-02 (omitted reasons become empty strings), D-03 (no server-side check of [N] markers in reasons), D-04 (range validation in the tool body, not schema annotations), D-05 (new frozen ScoreResult model in src/mcp_server/, src/models.py untouched), D-06 (response echoes weights and verdict thresholds), D-07 (optional domain string echoed verbatim).

Purpose: SCORE-01/02/03 are the core of milestone v1.2's hybrid grounding frame. The scoring math becomes grounding-by-construction: the server cannot return an arithmetically wrong total.
Output: src/mcp_server/scoring.py (new), score_account wired into src/mcp_server/server.py, tests/unit/test_mcp_scoring.py (new), extended functional and integration test files.
</objective>
